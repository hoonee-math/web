다른 사람의 도움을 받아서 소프트웨어를 만든다는 점은 비슷

library

도서관? 무언가가 정리정돈 되어있는 곳
즉, 내가 만들고자 하는 프로그램에 필요한 부품들이 되는 소프트웨어를 잘 정리정돈 해 놓은, 재사용하기 쉽도록 만들어둔 것이 라이브러리.
내가 만들고 있는 프로그램에 사용하는 부품을 가져오는 것

framework

만들고자 하는 것이 있을때, 만들고자 하는 것이 무엇이냐에 따라서(게임, 웹, 채팅 등) 그것을 만들려고 할때, 언제나 필요한 공통적인 부분이 있을 수 있고, 만들고자 하는 것에 대한 기획 의도에 따라서 달라지는 부분도 있을 수 있음.
공통적인 부분은 프레임워크라는 것을 만들어놓고, 만들고자 하는 기능이나 개성에 따라서 달라지는 부분만 살짝살짝 수정하는 것을 통해서 우리가 만들고자 하는 것을 처음부터 끝까지 만들지 않도록 해주는 반제품과 같은것이 프레임워크

library vs framework

라이브러리는 소프트웨어를 만드는 내가 라이브러리를 땡겨와서 쓰는 느낌이람면, 프레임워크는 프레임워크 안에 들어가서 작업하는 느낌.

js 대표 library: jQuery
